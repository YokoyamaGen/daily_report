# わかったこと
- Linux
  - Unixから派生したオープンソースのOS
  - メリット
    - 無料で使用できる
    - 品質の高いソフトウェア(MySQL、Apacheなど)を使用できる。
  - デメリット
    - 画像や動画、音声などの編集が苦手。
    - Micorsot Officeが動作しない
    - 日本語対応されていない部分が多い。

 - Linuxディストリビューション。カーネルだけだと機能が少ないので、使いやすいようにソフトウェアを組み込んだもの。広義のLinux
   - カーネル。OSの中核でハードウェアを制御する。狭義のLinux
   - ソフトウェア
   - 種類
     - Debian系。
        - Ubuntu
     - Red Hat系。
        - CentOS
        - Red Hat Enterprise Linux

  - Linuxの構成要素
    - ターミナル
      - ユーザにコマンド入力を提供するための画面。
    - シェル
      - ユーザにCLIを提供するソフトウェア。ユーザとカーネルの橋渡し役。ユーザが入力したコマンドをシェルが解釈して、カーネルに伝える。カーネルが実行した結果をシェルがユーザに伝える。
      - シェルの種類。
        - sh。最も古いシェル。
        - bash。Linuxのデフォルトシェル。shに機能を拡張したもの
        - zsh。bashに独自の機能を追加したもの。
    - カーネル

 - WindowsとLinuxとの違い
   - Windowsはディスク（CドライブやDドライブ）の下にフォルダーが作成される。一方、Linuxは1つのルートディレクトリの下にディレクトリが作成される。Linuxではディスクを利用する際には、ルートディレクトリの下にくっつけて利用する。そのことを **マウント** と呼ぶ。

- ディレクトリの種類
  - `bin`は一般ユーザおよび管理ユーザが使用するコマンドの実行ファイルを格納する場所。
  - `etc`は設定ファイルを格納する場所。
  - `var`は変化するデータを格納
  - `sbin`は管理者が使用するコマンドの実行ファイルを格納する場所。
  - `user`はライブラリなどを格納する場所。
  - `tmp`はプログラムの中で一時的に格納される場所。格納したファイルは消すことが多い。
  - `dev`はデバイスファイル(ハードウェアをファイルとして扱えるようにしたファイル)を格納する場所。

- よく使うコマンド、オプション
  - ディレクトリ操作
    - `ls -F`はファイル種別を表示する。`/`がディレクトリ、`@`はシンボリックリンク、`*`が実行可能ファイル、`記号なし`が通常ファイル
    - `rmdir`は空のディレクトリを削除する。空ではないディレクトリを削除しようとするとエラーがでる。
  - コマンドの調べ方
    - `man -k キーワード`でコマンドの候補を探す。
  - ファイル操作
    - `less`はファイルをスクロール表示する。ファイルの中身が多いときに使用する。
    - `ln　リンク元ファイル名　リンク名`はリンクを貼る。オプションを付けないと、ハードリンクが作成される。
      - `ls -s`でシンボリックリンクを作成。
      - 使用場面は長いパスを省略することや複数バージョンを共存させる時。シンボリックのほうが多く使う。
    - `find . -iname`はファイル名の大小を区別しない。`-name`は大小を区別する。

- ファイルの種類
  - テキストファイル
    - 文字コードで置き換えられた文字だけが書かれたファイルのこと。人間が理解できる文字が書かれたファイル。
    - 文字コードとは、Aが100001、Bは100010みたいに数字を文字として置き換えられている。
  - バイナリファイル
    - 文字コード以外の数値も含むファイルのこと。人間が理解できない文字が書かれたファイル。
    - 音声や動画、画像、実行ファイルなど

- リンクを貼る。ファイルに別名をつけること。
  - ハードリンク
    - 実体ファイルに別名をつけること。 **ファイルをコピーして別名にしているが、コピーとは違い、実体の変更した内容がハードリンクにも反映される。**
    - 実体を消しても、ハードリンクから中身を参照できる。
  - シンボリックリンク
    - 実体ファイルのパスの情報が格納されている。Windowsのショートカットみたいなもの。
    - 実体のファイルを消したり、移動させると、シンボリックリンクからは参照できない。

- 標準入出力。　**入力や出力をファイルやコマンドに変更できる＊＊
   - 標準入力。キーボードが使用される。
   - 標準出力。ディスプレイが使用される。
   - 標準エラー出力。ディスプレイが使用される。

- リダイレクト
  - 入力や出力を切り替えるためのもの。キーボードからファイルに入力を切り替えたり、出力を画面からファイルに切り替えたりする。
  - 標準入出力などをOSが識別するために用いる識別子。0が標準入力、1が標準出力、2が標準エラー出力。
  - `ls / /hoge > output.txt 2>&1`で標準入力および標準エラー出力がoutput.txtに出力されるようになる。

- プロセス
  - `ps aux`は`-`は不要。全ユーザの全プロセスを詳細まで表示する。

- ジョブ。まとめて実行されるプロセスの集まりをジョブ。`ps aux | grep bash | less`は3つのプロセスが生成されているが、ジョブだと1つ生成されている。
  - プロセスとの違い。プロセスはOS全体で管理されるが、ジョブはそのジョブを実行するシェルごとに管理される。
  - ジョブが使われるシーンとしてはバックグラウンドで処理する際にジョブIDを指定される。

- ジョブコマンド
  - `jobs`でジョブの一覧をみることができる。ジョブIDを調べられる。
  - `fg %ジョブID`でジョブをフォアグラウンドで動かせる。
  - `bg %ジョブID`でジョブをバックグラウンドで動かせる。
  - `コマンド　&`でコマンドをバックグラウンドで実行できる。
  - `Ctlr + z`でジョブを停止できる。

- ジョブやプロセスを終了させる
  - `kill %ジョブID`でジョブを終了させる
  - `kill プロセスID`でプロセスを終了させる
  - フォアグラウンドで実行しているジョブは`Ctrl + z`で終了させられる。
  - **バックグラウンドや停止中のジョブはkillコマンドで終了できる。**
  - `kill -9 プロセスID`で強制終了させられるが、オプションなそのkillコマンドで終了點せられなかったら、使用する。
