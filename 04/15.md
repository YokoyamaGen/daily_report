# わかったこと
- CSS
  - `float:left`を使用すると高さが0になり、それによって、レイアウトが崩れてしまうことを知りました。また、高さが0にするために、clearのdivタグやcssを書かなければいけないのは手間だと思い、複数の要素を横並びにするのにflexboxを使用しようと改めて思いました。
  - cssで*をクラス名に指定すると、全ての要素に適用することを初めて知りました。
  ```
   * {
      box-sizing: border-box;
   }
  ```
  - `box-sizing`を使用すると、paddingやborderの長さを含めて、widthを指定できることを学びました。あまり意識していなかったため、実務でも使っていきたいです。
  - flexboxでは複数の要素を横並びにしている要素を途中で折り返すことができることを知りました。レスポンシブデザインと組み合わせて使用されることも知りました。
  - flexboxでは複数の要素を横並びにしている要素を親要素の幅分だけ一杯に広げられることも知りました。

- Ruby
  - 変数の目的
    - データの中身が読み手にわかりやすいようにする。
    - 使いまわしがきくようにする。
    - プログラムの変更を簡単に対応できるようにする。

  - 複数のデータを格納
    - 配列
    - ハッシュ
      - キーがシンボル指定
      - キーが文字列指定

  - クラスは設計図。（レストランのメニューのような抽象的なもの。）
    - クラスメソッド。クラスをnewする前の状態で呼び出せるメソッド
      - 例えば、`Date.today`でメソッドを呼び出せる。クラスの中からもクラスメソッドを呼び出せる。
      - クラスメソッドを定義する際には、`def クラス名.メソッド名`
  - インスタンス。クラスから生成されたもの。（ピザやコーラなどの具体的なもの）
    - インスタンス変数
      - `attr_accessor シンボル`で定義する。
    - インスタンスメソッド
      - initialize
        - クラスからインスタンスを生成(new)した際に呼ばれるメソッド。インスタンス変数にデータを格納するのに使用されたりする。
        - インスタンス変数に値を格納したり、インスタンスメソッドから呼び出す際には`self`を変数の前につける。
  - `gets.chomp`はコンソール画面でユーザが入力した情報を受け取れる。
  - 継承
    - 子が親のインスタンス変数、インスタンスメソッドを利用できる。
    - オーバーライド
      - 親から継承したインスタンスメソッドの中身を子供側で変更できる。親メソッド自体は変更されない。
    - super
      - 親から継承したメソッドの中でsuperと書くことで、親の同名メソッドを使用できる。
